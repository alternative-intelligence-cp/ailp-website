Q Format: Fixed-Point Arithmetic Explained

What is Q Format?

Q Format = Qn.m (n integer bits, m fractional bits)

Binary number where decimal point is at fixed position: 

Q8.8 (8-bit integer, 8-bit fractional):


Binary:  0101 1010 . 1100 0011
^^^^^^^^   ^^^^^^^^
integer    fractional




Decimal: (0×128 + 1×64 + 0×32 + 1×16 + 1×8 + 0×4 + 1×2 + 0×1)
+ (1×0.5 + 1×0.25 + 0×0.125 + 0×0.0625 + ...)
= (90) + (0.7617...)
= 90.7617...

Key Insight: Decimal point position is convention, not stored in memory. Hardware does integer arithmetic; software interprets result as fixed-point.

fix256: Q128.128 Format

Total: 256 bits


Integer part: 128 bits (Q128.x)
Fractional part: 128 bits (Qx.128)




Binary layout:
[sign + 127-bit integer] . [128-bit fraction]
^^^^^^^^^^^^^^^^^^^^^^     ^^^^^^^^^^^^^^^^^
Range: ±2^127              Precision: 2^-128




Memory representation (4 × 64-bit limbs):
limb[3]: Bits 192-255 (most significant, contains sign)
limb[2]: Bits 128-191
limb[1]: Bits 64-127
limb[0]: Bits 0-63 (least significant, finest fraction)

Why Q128.128?

1. Symmetric range/precision: Equal bits for both sides of decimal point 2. Large range: ±2^127 ≈ ±1.7×10^38 (larger than universe in meters) 3. Ultra precision: 2^-128 ≈ 2.9×10^-39 (finer than Planck length!) 4. Multiplication simplicity: Q128.128 × Q128.128 = Q256.256 (middle 256 bits = Q128.128) 5. Division efficiency: Knuth's Algorithm D operates on 256-bit quotient

Alternative Q Formats (not used in Aria, but possible):

| Format | Integer Bits | Fractional Bits | Use Case | |--------|--------------|-----------------|----------| | Q64.192 | 64 | 192 | More precision, less range | | Q192.64 | 192 | 64 | More range, less precision | | Q127.129 | 127 | 129 | Asymmetric (complex implementation) |

Aria uses Q128.128 for balance and symmetry.
