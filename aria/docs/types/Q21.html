Q21 - Full-Spectrum Quantum Confidence with Superposition Certainty

Category: Types → Quantum/Cognitive Primitives Storage: tbb8 (Twisted Balanced Binary 8-bit) Range: -10 to +10 (21 discrete quantum states + ERR sentinel) Purpose: Granular confidence modeling for all quantum states including superposition and negation certainty Status: ✅ IMPLEMENTED (Phase 5.3 - February 2026) Philosophy: "Consciousness requires knowing not just WHAT you believe, but HOW CERTAIN you are - even about superposition itself"

Overview

Q21 extends Aria's quantum type family (Q3, Q9, Q21) to provide 5 confidence levels for ALL FOUR quantum states: NONE, OPTA, OPTB, and BOTH. Unlike Q9 where only options A and B have confidence gradations, Q21 treats superposition (BOTH) and negation (NONE) as first-class quantum states with their own certainty spectra. Critical Design Principle: In AGI consciousness substrates and multi-hypothesis tracking systems, being confident about superposition ("I'm CERTAIN the answer is simultaneously A AND B") or confident about negation ("I'm CERTAIN it's neither A nor B") is fundamentally different from being uncertain ("I don't know"). Q21 models this distinction with full granularity. 
// Q3: Binary choice + uncertainty


q3: Q3 = Q3::UNKNOWN;  // "I don't know"




// Q9: Confidence in A/B, binary BOTH/NONE
q9: Q9 = Q9::BOTH;     // "It's both, but I can't express certainty"




// Q21: Confidence in EVERYTHING including superposition
q21: Q21 = Q21(8);     // "I'm 60% confident it's BOTH states"
q21: Q21 = Q21(10);    // "I'm 100% confident it's superposition"
q21: Q21 = Q21(-10);   // "I'm 100% confident it's NONE of these"

Type Progression

Q3 → Q9 → Q21: Choosing Cognitive Resolution

Q3:  [A] ←→ [U] ←→ [B]


Simple binary choice with uncertainty (3 states)




Q9:  [Naaa] ←→ [U] ←→ [bbbB]
Adds confidence to A/B (3-5 levels)
BOTH and NONE exist but without confidence gradations
(~9 states depending on implementation)




Q21: [NNNNN] ←→ [aaaaa] ←→ [U] ←→ [bbbbb] ←→ [BBBBB]
Full 5-level confidence for ALL states (21 states)
Can express certainty about superposition and negation

Use Case Selection:
  • Q3: Simple yes/no/uncertain (obstacle left/right/unknown, on/off/transitioning)
  • Q9: Confidence in binary choices (sensor fusion, hypothesis strength)
  • Q21: Full superposition certainty (AGI reasoning, quantum measurement modeling, multi-hypothesis tracking with confidence in "it's BOTH")

Storage Format

struct Q21 {


tbb8:value;  // Symmetric range: -127 to +127, ERR = -128
// Uses: -10 to +10 for quantum states
}

Why tbb8 (not int8):
  • Symmetric around zero: -127 to +127 (vs int8: -128 to +127)
  • ERR sentinel: -128 is sticky error state (propagates through operations)
  • Semantic clarity: TBB designed for symmetric confidence modeling
  • Zero-centered: UNKNOWN naturally sits at 0

Key Design Features

1. Saturation Barriers (±6)

Purpose: Prevent accidental drift from option confidence into superposition/negation 
// Max confidence B saturates before BOTH range


q: Q21 = Q21(5);   // OPTB_MAX (100% confident it's B)
q++;               // Still Q21(5) - saturates, doesn't drift to BOTH
q++;               // Still Q21(5) - intentional barrier




// Must explicitly transition to BOTH
q = Q21(6);        // BOTH_MIN - intentional semantic shift

Rationale: "I'm maximally confident it's option B" is semantically DIFFERENT from "I'm weakly confident it's BOTH options". The gap at ±6 forces intentional transitions.

2. Crystallization Thresholds (±3, ±8)

Purpose: State can only collapse to definite measurement when confidence ≥60% 
// Confidence too weak to crystalize


q: Q21 = Q21(2);          // OPTB_LOW (40% confident)
result: bool = q.crystalize_to_B();  // fail(ERR) - confidence < PERC_LVL_OPTB




// Sufficient confidence to crystalize
q: Q21 = Q21(3);          // OPTB_MED (60% confident)
result: bool = q.crystalize_to_B();  // pass(true) - can collapse




// Crystallization thresholds:
// PERC_LVL_OPTA = -3 (60% confidence A)
// PERC_LVL_OPTB = +3 (60% confidence B)
// PERC_LVL_NONE = -8 (60% confidence NONE)
// PERC_LVL_BOTH = +8 (60% confidence BOTH)

Rationale: Quantum measurement should only collapse when evidence is strong (≥60%). Prevents premature collapse from noise.

3. Symmetric Confidence Encoding

ALL states (NONE, A, B, BOTH) have identical 5-level confidence structure: 
MIN:  20% confidence (can saturate here from below)


LOW:  40% confidence
MED:  60% confidence (crystallization threshold)
HIGH: 80% confidence
MAX:  100% confidence (can saturate here from above)

This symmetry enables uniform confidence reasoning across all quantum states.

Operations

Construction

// From confidence value


q: Q21 = Q21(0);      // UNKNOWN
q: Q21 = Q21(5);      // OPTB_MAX (100% B)
q: Q21 = Q21(-10);    // NONE_MAX (100% NONE)
q: Q21 = Q21(10);     // BOTH_MAX (100% BOTH)




// Named constructors
q: Q21 = Q21::UNKNOWN;
q: Q21 = Q21::OPTB_MAX;
q: Q21 = Q21::NONE_HIGH;
q: Q21 = Q21::BOTH_MED;




// From boolean with max confidence
q: Q21 = Q21::from_bool(false);  // OPTA_MAX (-5)
q: Q21 = Q21::from_bool(true);   // OPTB_MAX (+5)

Confidence Adjustment

q: Q21 = Q21(3);   // OPTB_MED (60% confident B)


q++;               // Q21(4) - OPTB_HIGH (80%)
q++;               // Q21(5) - OPTB_MAX (100%)
q++;               // Q21(5) - saturates at barrier




q--;               // Q21(4) - back to 80%




// Large adjustments
q += 3;            // Increase confidence by 3 levels
q -= 5;            // Decrease confidence by 5 levels (with saturation)

State Queries

q: Q21 = Q21(8);


// Basic state
q.is_OPTB();       // false (not in B range)
q.is_BOTH();       // true (in BOTH range)
q.is_UNKNOWN();    // false
q.is_NONE();       // false




// Confidence level
q.confidence();    // 60 (60% confidence)
q.is_certain();    // false (certainty requires ±10)
q.above_threshold(); // true (≥ PERC_LVL_BOTH)




// Range queries
q.is_superposition();  // true (BOTH range)
q.is_negation();      // false (NONE range)
q.is_option_A();      // false (OPTA range)
q.is_option_B();      // false (OPTB range)

Crystallization (Measurement Collapse)

// Collapse to boolean (requires confidence ≥60%)


q: Q21 = Q21(4);           // OPTB_HIGH (80%)
result: Result = q.crystalize();
// pass(true) - collapsed to B with high confidence




// Collapse to specific option
q: Q21 = Q21(-3);          // OPTA_MED (60%)
result: Result = q.crystalize_to_A();
// pass(true) - sufficient confidence




q: Q21 = Q21(2);           // OPTB_LOW (40%)
result: Result = q.crystalize_to_B();
// fail(ERR) - insufficient confidence




// Superposition/negation collapse requires ≥60% confidence
q: Q21 = Q21(8);           // BOTH_MED (60%)
can_collapse: bool = q.can_crystalize_BOTH();  // true




q: Q21 = Q21(-8);          // NONE_MED (60%)
can_collapse: bool = q.can_crystalize_NONE();  // true

Use Cases

AGI Consciousness Substrate

Problem: AGI reasoning often produces confident superposition states - "I'm CERTAIN the answer is both 'morally correct' AND 'pragmatically optimal'" vs "I'm UNCERTAIN which is right" 
struct:MoralJudgment = {


Q21:is_action_justified;  // Can be BOTH with high confidence
}




fn evaluate_trolley_problem() -> MoralJudgment {
// Utilitarian: Save 5 people (B = justified)
utilitarian: Q21 = Q21(4);  // 80% confident justified




// Deontological: Don't actively kill (A = not justified)
deontological: Q21 = Q21(-4);  // 80% confident not justified




// AGI realizes BOTH frameworks are simultaneously valid
// This is NOT uncertainty - it's confident superposition
judgment: MoralJudgment = MoralJudgment{
is_action_justified: Q21(9)  // 80% confident it's BOTH
};




return judgment;
}




// Later: Can crystalize if forced to act
action: Result = judgment.is_action_justified.crystalize();
// Collapses superposition based on context/utilities

Key: Q21 distinguishes "I'm confident it's BOTH" (superposition certainty) from "I don't know which" (uncertainty).

Quantum Sensor Fusion

Problem: Multiple sensors measuring same phenomenon may confidently disagree (both are correct due to measurement context) 
struct:SensorReading = {


Q21:object_detected;
}




fn fuse_sensors(infrared: bool, radar: bool, camera: bool) -> SensorReading {
reading: Q21 = Q21::UNKNOWN;




when infrared then reading += 2; end  // Weak evidence for B
when radar then reading += 3; end     // Moderate evidence for B
when camera then reading -= 2; end    // Weak evidence for A




// If all sensors agree (conflicting readings)
when infrared && radar && !camera then
// BOTH sensors (IR, radar) confidently detect
// Camera confidently doesn't detect
// This is superposition, not error!
reading = Q21(8);  // BOTH_MED - confident superposition
end




return SensorReading{object_detected: reading};
}




// Later: Decide action based on confidence
when sensor.object_detected.is_BOTH() &&
sensor.object_detected.confidence() >= 60 then
// Take conservative action - treat as obstacle
brake();
end

Multi-Hypothesis Tracking

Problem: Tracking systems may be certain multiple hypotheses are simultaneously valid (not uncertain which is right) 
struct:TrackHypothesis = {


Q21:target_is_hostile;
}




fn update_track(track: TrackHypothesis, new_data: SensorData) -> TrackHypothesis {
// Friendly IFF signal detected
when new_data.has_friendly_IFF then
track.target_is_hostile -= 3;  // Shift toward A (not hostile)
end




// But also: Radar signature matches known threats
when new_data.matches_threat_signature then
track.target_is_hostile += 3;  // Shift toward B (hostile)
end




// Electronic countermeasures detected (spoofing possible)
when new_data.has_ECM then
// BOTH states likely: It's friendly AND spoofed to look hostile
// OR: It's hostile AND spoofed to look friendly
// This is confident superposition, not uncertainty!
track.target_is_hostile = Q21(7);  // BOTH_LOW
end




return track;
}




// Decision under superposition certainty
when track.target_is_hostile.is_BOTH() &&
track.target_is_hostile.above_threshold() then
// Query human operator - confident it's BOTH states
request_human_judgment();
else when track.target_is_hostile.can_crystalize_to_B() then
// Sufficient confidence it's hostile
engage_defenses();
end

Negation Certainty (NONE Range)

Problem: Sometimes you're certain the answer is neither option - not just uncertain 
struct:LanguageDetection = {


Q21:is_english_or_spanish;  // A = English, B = Spanish
}




fn detect_language(text: string) -> LanguageDetection {
detection: Q21 = Q21::UNKNOWN;




// Analyze character frequency
when has_mandarin_characters(text) then
// Confident it's NEITHER English NOR Spanish
detection = Q21(-9);  // NONE_HIGH (80% confident NONE)
else when has_english_words(text) then
detection += 3;  // Shift toward B (Spanish) or A (English)
end




return LanguageDetection{is_english_or_spanish: detection};
}




// Handle negation separately from uncertainty
when lang.is_english_or_spanish.is_NONE() &&
lang.is_english_or_spanish.above_threshold() then
// Confidently neither - use different language model
switch_to_CJK_model();
else when lang.is_english_or_spanish.is_UNKNOWN() then
// Genuinely uncertain - need more data
request_more_samples();
end

Comparison: Q3 vs Q9 vs Q21

Storage & Range

| Type | Storage | States | Range | ERR Sentinel | |------|---------|--------|-------|--------------| | Q3 | trit | 3 | A, U, B | magic trit state | | Q9 | nit | 9 | NONE, 5×A, U, 5×B, BOTH | magic nit state | | Q21 | tbb8 | 21 | 5×NONE, 5×A, U, 5×B, 5×BOTH | -128 |

Feature Matrix

| Feature | Q3 | Q9 | Q21 | |---------|----|----|-----| | Binary options (A/B) | ✅ | ✅ | ✅ | | Uncertainty (UNKNOWN) | ✅ | ✅ | ✅ | | Confidence in A | ❌ | ✅ (5 levels) | ✅ (5 levels) | | Confidence in B | ❌ | ✅ (5 levels) | ✅ (5 levels) | | Superposition (BOTH) | ❌ | ✅ (binary) | ✅ (5 levels) | | Negation (NONE) | ❌ | ✅ (binary) | ✅ (5 levels) | | Superposition certainty | ❌ | ❌ | ✅ | | Negation certainty | ❌ | ❌ | ✅ | | Crystallization threshold | ❌ | Partial | ✅ (all states) | | Saturation barriers | ❌ | Partial | ✅ (±6 gaps) |

Performance

// Memory footprint


sizeof(Q3):  1 trit  (~0.33 bytes, implementation-dependent)
sizeof(Q9):  1 nit   (~0.67 bytes, implementation-dependent)
sizeof(Q21): 1 byte  (tbb8)




// Cache efficiency: Q21 is best (fits standard 8-bit operations)




// Operations (all O(1)):
Q3:  Comparison, crystalization
Q9:  Comparison, confidence queries, crystalization
Q21: Comparison, confidence queries, crystallization, saturation checks

When to Use Each

Use Q3 when:
  • Simple binary choice with uncertainty
  • Memory extremely constrained (sub-byte storage)
  • No confidence modeling needed
  • Example: "Is obstacle left/right/unknown?"
Use Q9 when:
  • Confidence in binary options matters
  • Superposition/negation exist but as binary states
  • Don't need certainty about superposition itself
  • Example: "How confident are we sensor A or B is right?"
Use Q21 when:
  • Need confidence in superposition ("certain it's BOTH")
  • Need confidence in negation ("certain it's NEITHER")
  • AGI reasoning with confident multi-hypothesis states
  • Quantum measurement modeling with collapse thresholds
  • Example: "How certain am I that BOTH moral frameworks apply?"

Edge Cases

Saturation at Barriers

q: Q21 = Q21(5);   // OPTB_MAX


q += 10;           // Tries to reach +15
// Result: Q21(5) - saturates at barrier, doesn't enter BOTH range




q: Q21 = Q21(-6);  // NONE_MIN
q++;               // Tries to reach -5
// Result: Q21(-6) - saturates, doesn't cross into OPTA range

Rationale: Prevents semantic drift. Moving from "confident B" to "confident BOTH" requires intentional state change.

ERR Propagation

q1: Q21 = Q21::ERR;


q2: Q21 = Q21(5);




q3: Q21 = q1 + 1;  // Still ERR (sticky)
q4: Q21 = q2 + q1; // ERR (propagates)




// Check for ERR before operations
when q1.is_ERR() then
log("Error in quantum state!");
end

Crystallization from UNKNOWN

q: Q21 = Q21::UNKNOWN;  // Confidence = 0%


// Can't crystalize with zero confidence
result: Result = q.crystalize();
// fail(ERR) - no evidence for any state




// Must accumulate evidence first
q += 3;  // Now OPTB_MED (60%)
result: Result = q.crystalize();
// pass(true) - sufficient confidence

Symmetric Confidence Decay

// Confidence decayst symmetrically


q: Q21 = Q21(10);  // BOTH_MAX (100% superposition)




// Over time: Evidence weakens
q--;  // BOTH_HIGH (80%)
q--;  // BOTH_MED (60%)
q--;  // BOTH_LOW (40%)
q--;  // BOTH_MIN (20%)
q--;  // BOTH_MIN (saturates at barrier, can't decay to OPTB)




// Must explicitly transition
q = Q21(5);  // Now OPTB_MAX (left superposition intentionally)

Best Practices

✅ DO: Use Confidence Levels Meaningfully

// Good: Confidence reflects evidence strength


evidence_count: int32 = 0;
q: Q21 = Q21::UNKNOWN;




till(sensors.length - 1, 1) {
when sensors[$].detects_B() then
evidence_count++;
q += 1;  // Increment confidence per evidence
end
}




// Result: q.confidence() proportional to evidence

✅ DO: Check Crystallization Threshold

// Good: Only collapse when confident enough


when q.above_threshold() then
decision: bool = q.crystalize()?;  // Safe to collapse
else
log("Insufficient confidence, need more data");
collect_more_evidence();
end

✅ DO: Distinguish Superposition from Uncertainty

// Good: Different handling for BOTH vs UNKNOWN


when q.is_BOTH() && q.confidence() >= 60 then
// Confidently both - handle superposition
handle_superposition_case();
else when q.is_UNKNOWN() then
// Genuinely uncertain - gather data
request_more_sensors();
end

❌ DON'T: Ignore Saturation Barriers

// Wrong: Assuming linear drift


q: Q21 = Q21(5);   // OPTB_MAX
q++;               // Still 5 (saturated)
when q == Q21(6) then  // ❌ Never true!
handle_BOTH();
end




// Right: Explicitly transition
q = Q21(6);  // Intentional move to BOTH range
when q.is_BOTH() then handle_BOTH(); end

❌ DON'T: Crystalize Without Threshold Check

// Dangerous: Might fail


q: Q21 = Q21(2);  // Only 40% confident
decision: bool = q.crystalize()?;  // ❌ Will fail!




// Safe: Check first
when q.can_crystalize() then
decision: bool = q.crystalize()?;
else
log("Cannot crystalize - confidence too low");
end

❌ DON'T: Treat NONE as UNKNOWN

// Wrong: Conflating negation with uncertainty


when q.is_NONE() then
// Thinking: "Don't know, need more data"
request_more_data();  // ❌ Wrong! NONE means "confident it's neither"
end




// Right: Separate handling
when q.is_NONE() && q.above_threshold() then
// Confident it's neither A nor B - try different options
explore_alternative_hypotheses();
else when q.is_UNKNOWN() then
// Genuinely uncertain - need data for A/B decision
request_more_data();
end

Q21 is quantum reasoning with full confidence granularity. It enables AGI systems to express:
  • "I'm CERTAIN the answer is both A AND B" (confident superposition)
  • "I'm CERTAIN the answer is neither A nor B" (confident negation)

By providing 5 confidence levels for all four quantum states (NONE, A, B, BOTH), Q21 supports nuanced cognitive modeling where:

  • Superposition is not just binary (BOTH vs not-BOTH), but has degrees of certainty
  • Negation is not just absence of evidence, but confident rejection
  • Evidence accumulation naturally increases confidence within each state range
  • Saturation barriers prevent accidental semantic drift between fundamentally different quantum states
Use Q21 when your AGI, sensor fusion, or multi-hypothesis system needs to reason about the certainty of superposition itself - not just which option, but how confident you are that BOTH options simultaneously apply.
