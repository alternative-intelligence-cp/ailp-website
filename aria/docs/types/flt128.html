The flt128 Type (Quadruple Precision Floating Point)

Category: Types → Floating Point → Extended Precision Syntax: flt128 (alias: f128) Purpose: 128-bit quadruple-precision floating point for extreme scientific accuracy First Available: Aria v0.0.1 Status: ✅ Fully implemented (software emulation)

Overview

flt128 is Aria's quadruple-precision floating-point type, providing approximately 34 decimal digits of precision with a massive range (±10⁴⁹³²). This is DOUBLE the precision of double (15 digits) and FIVE TIMES the precision of float (7 digits). Key Characteristics:
  • 128 bits (16 bytes): 1 sign + 15 exponent + 112 mantissa bits
  • ~34 decimal digits of precision
  • Range: ±1.2 × 10⁴⁹³² (cosmological scale!)
  • Deterministic: Bit-exact results across platforms
  • Software emulation: No native CPU hardware (very slow!)
  • ~50-100× slower than double
Alias: f128 When to use:
  • ✅ Extreme precision scientific computing (quantum mechanics, astrophysics)
  • ✅ Long-running simulations where error accumulation matters
  • ✅ Numerical stability requirements beyond double
  • ✅ Bridging gap between double and arbitrary precision
When NOT to use:
  • ❌ Real-time applications (too slow)
  • ❌ Financial calculations (use fix256 for EXACT decimals)
  • ❌ General purpose (use double - 15 digits sufficient)
  • ❌ Memory-constrained systems (16 bytes per value)
  • ❌ SIMD/GPU (no hardware support)

Specifications

| Aspect | flt128 | double | float | |--------|--------|--------|-------| | Size | 16 bytes | 8 bytes | 4 bytes | | Precision | ~34 digits | ~15 digits | ~7 digits | | Range | ±10⁴⁹³² | ±10³⁰⁸ | ±10³⁸ | | Epsilon | ~1.9×10⁻³⁴ | ~2.2×10⁻¹⁶ | ~1.2×10⁻⁷ | | Speed | 50-100× slow | 1× | 2× (SIMD) | | Hardware | ❌ Software | ✅ Native | ✅ Native | | Deterministic | ✅ Yes | ❌ Varies | ❌ Varies |

Declaration

// Explicit suffix REQUIRED (zero implicit conversion policy)


flt128:pi = 3.14159265358979323846264338327950288f128;




// Alias
f128:e = 2.71828182845904523536028747135266250f128;




// Scientific notation
f128:avogadro = 6.02214076e23f128;
f128:planck = 6.62607015e-34f128;




// ❌ WRONG - missing suffix
flt128:x = 0;      // COMPILE ERROR
flt128:y = 3.14;   // COMPILE ERROR

Rationale: Child-safety - no hidden assumptions.

Precision & Range

34 Decimal Digits

// Full precision


flt128:precise = 1.234567890123456789012345678901234f128;
//                ↑ 34 significant digits




// vs double (only ~15 digits)
double:limited = 1.2345678901234567;  // Last digits unreliable




// Precision test
flt128:a = 1.0e20f128 + 1.0f128;
// Result: 100000000000000000001.0 (all 34 digits!)




double:b = 1.0e20 + 1.0;
// Result: 100000000000000000000.0 (precision lost)

Massive Range

// Quantum scale


flt128:electron_mass = 9.1093837015e-31f128;  // kg




// Cosmological scale
flt128:universe_meters = 8.8e26f128;
flt128:atoms_in_universe = 1.0e80f128;




// Extreme (no overflow until ~10⁴⁹³²!)
flt128:huge = 1.0e4000f128;    // OK
flt128:overflow = 1.0e5000f128; // INF

Arithmetic

flt128:a = 100.0f128;


flt128:b = 3.0f128;




flt128:sum = a + b;       // 103.0 (precise to 34 digits)
flt128:diff = a - b;      // 97.0
flt128:product = a * b;   // 300.0
flt128:quotient = a / b;  // 33.333333... (34 digits)
flt128:remainder = a % b; // 1.0




// Division by zero
flt128:inf = 10.0f128 / 0.0f128;  // +INF
flt128:nan = 0.0f128 / 0.0f128;   // NaN

Performance: All operations ~50-100× slower than double (software emulation).

Special Values

Signed Zero

flt128:pos_zero = 0.0f128;   // +0.0


flt128:neg_zero = -0.0f128;  // -0.0




if (pos_zero == neg_zero) {
// TRUE - but bit patterns differ!
}

Infinity

const flt128:INF = 1.0f128 / 0.0f128;


const flt128:NEG_INF = -INF;




flt128:r1 = INF + 1000.0f128;  // +INF
flt128:r2 = INF - INF;         // NaN (undefined)

NaN

const flt128:NAN = 0.0f128 / 0.0f128;


// NaN != NaN is TRUE!
if (NAN != NAN) {
stdout.write("NaN ≠ NaN\n");  // Fundamental property
}




// NaN propagation
flt128:r = NAN + 10.0f128;  // NaN




// Check for NaN
func:is_nan = bool(flt128:value) {
pass(value != value);
};

Determinism

Aria's flt128 is fully deterministic (unlike IEEE 754 float/double): 
// GUARANTEED bit-exact across ALL platforms


flt128:a = 0.1f128;
flt128:b = 0.2f128;
flt128:c = a + b;  // EXACTLY 0.3f128 everywhere




// Compare IEEE double (may vary)
double:x = 0.1;
double:y = 0.2;
double:z = x + y;  // Platform-dependent!

Platforms: x86_64, ARM64, RISC-V, all OS, all compilers → identical results.

Comparisons

⚠️ DANGER: Direct equality is still risky (even with 34 digits)! 

// ANTI-PATTERN


flt128:a = 1.0f128 / 3.0f128;
flt128:b = a * 3.0f128;
if (b == 1.0f128) {  // Might be FALSE!
}




// ✅ CORRECT: Epsilon comparison
const flt128:TOL = 1.0e-30f128;




func:almost_equal = bool(flt128:a, flt128:b, flt128:eps) {
flt128:diff = a - b;
if (diff < 0.0f128) { diff = -diff; }
pass(diff < eps);
};




if (almost_equal(b, 1.0f128, TOL)) {
stdout.write("Close enough!\n");
}

Type Conversions

Widening (Safe)

// float → flt128 (exact)


float:small = 3.14;
flt128:wide = flt128(small);




// double → flt128 (exact)
double:medium = 2.718281828459045;
flt128:wider = flt128(medium);

Narrowing (Precision Loss!)

// flt128 → double (LOSES precision beyond ~15 digits)


flt128:precise = 3.14159265358979323846264338327950288f128;
double:rounded = double(precise);  // ~3.141592653589793




// Range check before narrowing
flt128:huge = 1.0e400f128;
if (huge > flt128(DBL_MAX)) {
stderr.write("Too large for double!\n");
}

Performance

~50-100× SLOWER than double: 
// Benchmark: 1M operations


// double:  ~2ms   (native hardware)
// flt128:  ~100-200ms (software emulation)




flt128:sum = 0.0f128;
for (tbb32:i = 0i32; i < 1000000i32; ++i) {
sum += flt128(i);  // SLOW
}

Memory: 16 bytes per value (2× double, 4× float) Cache: Holds half as many values per cache line

Use Cases

Astronomy

// Cosmological timescales


flt128:age_universe = 4.35e17f128;  // seconds




// Extreme distances
flt128:observable_radius = 8.8e26f128;  // meters

Quantum Mechanics

const flt128:PLANCK_H = 6.62607015e-34f128;


const flt128:ALPHA = 7.2973525693e-3f128;  // Fine structure

Long Simulations

// Million-timestep simulation (double would drift)


flt128:position = 0.0f128;
flt128:velocity = 1.0e-10f128;
const flt128:dt = 1.0e-6f128;




for (tbb32:step = 0i32; step < 1000000i32; ++step) {
position += velocity * dt;  // 34-digit precision
}
// Still accurate after 1M steps!

Anti-Patterns

❌ Default Use

// WRONG: Overkill


flt128:counter = 0.0f128;  // ❌ use int32!
flt128:percent = 0.5f128;  // ❌ use float!




// CORRECT: Match precision to need
int32:counter = 0i32;
float:percent = 0.5;

❌ Financial Math

// WRONG: Still not exact!


flt128:price = 19.99f128;
flt128:total = price * 3.0f128;  // ❌ 59.97000...




// CORRECT: Exact decimals
fix256:price = fix256("19.99");
fix256:total = price * fix256("3");  // Exactly 59.97

❌ Hot Loops

// WRONG: Billion ops at 100× slowdown!


for (tbb32:i = 0i32; i < 1000000i32; ++i) {
for (tbb32:j = 0i32; j < 1000i32; ++j) {
flt128:r = flt128(i) * flt128(j);  // ❌ TOO SLOW
}
}




// CORRECT: Use double, promote to flt128 at end
for (...) {
double:r = double(i) * double(j);
accumulator += r;
}
flt128:final = flt128(accumulator);

Common Patterns

High-Precision Constants

const flt128:PI = 3.14159265358979323846264338327950288f128;


const flt128:E = 2.71828182845904523536028747135266250f128;
const flt128:SQRT2 = 1.41421356237309504880168872420969808f128;

Safe Narrowing

func:safe_to_double = Result(flt128:value) {


if (value > flt128(DBL_MAX)) { fail("Overflow"); }
if (value < flt128(-DBL_MAX)) { fail("Underflow"); }
pass(double(value));
};

Kahan Summation

func:kahan_sum = flt128(flt128[]:values, tbb32:count) {


flt128:sum = 0.0f128;
flt128:c = 0.0f128;  // Compensation




for (tbb32:i = 0i32; i < count; ++i) {
flt128:y = values[i] - c;
flt128:t = sum + y;
c = (t - sum) - y;  // Capture lost bits
sum = t;
}
pass(sum);
};

vs Other Types

vs double

  • Precision: 34 digits vs 15
  • Speed: 50-100× slower
  • Size: 16 bytes vs 8
  • Use when: Need >15 digits

vs fix256

  • Type: Floating vs Fixed
  • Exact decimals: ❌ No vs ✅ Yes
  • Range: ±10⁴⁹³² vs ±10³⁸
  • Use for: Science vs Finance

vs tfp64

  • Precision: 34 digits vs 14
  • Both deterministic: ✅ Yes
  • Speed: 50-100× vs 5-10× slower
  • Use when: Need >14 digits deterministic

// 100-hour session (360M timesteps at 1kHz)


flt128:emotional_state = 0.0f128;
const flt128:dt = 0.001f128;




for (tbb64:step = 0i64; step < 360000000i64; ++step) {
flt128:stimulus = therapy_stimulus(step);
emotional_state += stimulus * dt;
}
// Accurate after 360M iterations!

9D Wave Interference

struct:Wave9D {


complex[9]:amplitudes;
};




func:interfere = flt128(Wave9D:a, Wave9D:b) {
flt128:total = 0.0f128;
for (tbb8:dim = 0i8; dim < 9i8; ++dim) {
complex:sum = a.amplitudes[dim] + b.amplitudes[dim];
total += sum.magnitude_squared();
}
pass(sqrt(total));
};

struct:Example {

};
// Total: 32 bytes

C/FFI

// Aria flt128 ↔ C __float128 (GCC/Clang)


extern "C" {
func:c_sqrt_f128 = flt128(flt128:x) -> "__float128 quadmath_sqrt(__float128)";
};




flt128:result = c_sqrt_f128(2.0f128);

Platform Differences

NO hardware support on ANY platform:
  • x86_64: Software only
  • ARM64: Software only
  • RISC-V: Software only
  • GPU: No support
Result: Consistent performance (~50-100× slower everywhere). Determinism: Bit-exact across all platforms, compilers, optimization levels.

Related

  • float - 32-bit, ~7 digits, fast, non-deterministic
  • double - 64-bit, ~15 digits, fast, non-deterministic
  • flt256 - 256-bit, ~70 digits, very slow
  • flt512 - 512-bit, ~150 digits, extremely slow
  • tfp64 - Deterministic 64-bit, ~14 digits
  • fix256 - Exact decimals (finance/legal)
Remember:

1. 50-100× SLOWER than double - use only when >15 digits needed 2. Still approximates - NOT exact for 0.1, 0.2 (use fix256 for finance!) 3. Deterministic - bit-exact across platforms (unlike float/double) 4. No hardware - software emulation everywhere 5. Epsilon comparisons - never direct x == y 6. Massive range - ±10⁴⁹³² (cosmological scale) 7. 16 bytes - 2× memory of double

When to prefer alternatives:
  • General use: double (15 digits sufficient)
  • Exact decimals: fix256/frac64 (finance)
  • Fast deterministic: tfp64 (5-10× slower)
  • Real-time: float/double (hardware critical)
